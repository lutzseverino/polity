# Tutorials

Tutorials teach a complete outcome by guiding the reader through it.

## Use This Section For

- onboarding paths for unfamiliar contributors
- guided exercises that build understanding through a complete outcome
- learning material where context and sequence matter

## Do Not Use This Section For

- one-off setup steps; use [How-To Guides](../how-to/README.md)
- factual contracts or lookup tables; use [Reference](../reference/README.md)
- architectural rationale; use [Explanation](../explanation/README.md)
- ephemeral notes that are not intended as durable documentation

## Required Shape

Every tutorial should include:

- `Goal`
- `Prerequisites`
- `Steps`
- `Result`

Use the [tutorial template](../_templates/tutorial.template.md) for new tutorials.
