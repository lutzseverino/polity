# Decisions

Decision records capture durable choices with meaningful tradeoffs.

## Use This Section For

- architecture decisions
- boundary decisions
- convention decisions
- rejected alternatives that future contributors are likely to reconsider

## Do Not Use This Section For

- draft proposals that have not become durable decisions
- factual contracts; use [Reference](../reference/README.md)
- broad conceptual background; use [Explanation](../explanation/README.md)
- implementation checklists

## Required Shape

Decision records must use this section order:

- `Status`
- `Context`
- `Decision`
- `Consequences`
- `Alternatives Considered`

Use the [decision template](../_templates/decision.template.md) for new decision records.
